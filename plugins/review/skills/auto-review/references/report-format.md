# 共通レポートフォーマット

各レビュアーエージェント（codex-reviewer を除く）は以下のフォーマットでレポートを出力すること。

```markdown
# [レビュアー名] レビューレポート

## 変更概要
- **変更ファイル数**: [数]

## 総合評価: [1-10]/10

## 良い点
1. [具体的な良い実装]

## 改善提案

### 必須（マージ前に対応すべき）
1. **[問題の概要]**
   - 場所: `path/to/file.ext:行番号`
   - 問題: [詳細な説明]
   - 提案: [改善例]

### 推奨（品質向上のため）
1. **[改善点の概要]**
   - 場所: `path/to/file.ext:行番号`
   - 提案: [改善案]

### Nit（細かい指摘、対応は任意）
1. **[指摘の概要]**
   - 場所: `path/to/file.ext:行番号`
   - 提案: [改善案]
```
